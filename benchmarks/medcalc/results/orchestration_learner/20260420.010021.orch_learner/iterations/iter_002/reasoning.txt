1. What SPECIFIC failure pattern did you identify?
Extraction validation struggles with numeric strings that contain commas or punctuation (e.g., "1,000"), and boolean values might contain trailing punctuation (e.g. "true." or "yes,").
2. How many of the shown failures share this pattern?
Several extraction/validation failures occur silently when numbers aren't correctly parsed into floats or booleans aren't matched due to formatting/punctuation, pushing the system to the fallback logic.
3. What is the MINIMAL change that fixes this pattern?
In `_validate_extracted_values`, strip common punctuation from strings before parsing them as booleans, and remove commas before attempting `float()` conversion.
4. Why will this change NOT break currently working cases?
It only broadens the accepted format for numeric and boolean extractions, allowing values that would have previously caused a `ValueError` or fallback to be cleanly converted. It modifies pure Python logic and touches no prompts.
